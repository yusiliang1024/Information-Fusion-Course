# 模型与处理结构
## JDL（Joint Directors of Laboratories）数据融合功能模型：JDL 模型的核心是从 **低级处理（数据驱动）向高级处理（认知驱动）** 演进的过程。
### L1: 目标评估 (Object Assessment)
- 消化核心： 解决“是什么”和“在哪里”的问题。
  - 关键任务： * 多传感器对准：坐标转换、时间同步；数据关联：目标位置和运动学、属性参数、身份估计。用于提供辅助决策信息* 。
  - 属性判别实例：
    - 雷达目标分类输入数据：雷达回波的 RCS（雷达散射截面积）序列。
    - 处理逻辑：如果 $RCS > 100m^2$，且速度在 $250m/s$ 左右 $\rightarrow$ 识别为：民航客机。如果 $RCS \approx 2m^2$，且伴随高机动过载（大转弯） $\rightarrow$ 识别为：战斗机。
    - 融合点：若此时红外传感器（IR）检测到强烈的热源信号，则进一步确认为“开着加力燃烧室的战斗机”。比如通过雷达 RCS 面积判断是民航机还是战斗机。
  - 数据关联：机场跑道多目标跟踪场景：两架飞机在跑道上交叉滑行。
    - 挑战：传感器在 $T_1$ 时刻看到两个点，在 $T_2$ 时刻也看到两个点。
    - 任务：确定 $T_2$ 的点 A 到底是由 $T_1$ 的哪架飞机移动过来的？
    - 算法应用：使用 GNN（全局最近邻） 或 JPDA（联合概率数据关联） 算法，通过计算预测位置与实际观测位置的“马氏距离”来决定连线方式。
- 数学工具： 主要是你熟悉的卡尔曼滤波（KF）、多目标跟踪算法（全局最近邻算法 (GNN, Global Nearest Neighbor), 联合概率数据关联 (JPDA, Joint Probabilistic Data Association)）
  ### L1 层级核心算法对比

| 算法 | 解决的核心问题 | 核心思想 | 适用场景 |
| :--- | :--- | :--- | :--- |
| **KF (卡尔曼滤波)** | **状态估计** | 预测 + 观测修正。通过计算 Kalman Gain，在模型预测和传感器观测之间寻找最优权重。 | 单目标跟踪、传感器基础数据融合。 |
| **GNN (全局最近邻)** | **点迹与目标关联** | “非黑即白”的硬关联。计算所有观测点与目标预测点的马氏距离，寻求全局总距离最短的匹配方案。 | 目标稀疏、背景干净（杂波少）、目标不易交叉的场景。 |
| **JPDA (联合概率关联)** | **复杂环境关联** | “概率性”的软关联。认为一个观测点可能属于多个目标，计算所有可能的关联概率并进行加权。 | 密集多目标跟踪、高杂波干扰、目标轨迹频繁交叉的场景。 |

### L2: 態勢評估 (Situation Assessment)

- 1. 核心邏輯
  - **解決問題**：解決「發生了什麼？」以及「實體間的關係是什麼？」。
  - **演進特徵**：從 L1 的單體目標處理，轉向對**目標群體、環境、背景**的綜合理解。

- 2. 關鍵任務
  - **實體聚合 (Aggregation)**：將多個單體目標看作一個邏輯整體（例如：多架飛機 $\rightarrow$ 一個飛行編隊）。
  - **關係提取 (Relation Extraction)**：
    * **空間關係**：目標是否在敏感區域？目標間的相對位置（側翼、包圍）。
    * **因果關係**：目標 A 的機動是否是為了配合目標 B 的攻擊？
  - **上下文推理 (Contextual Reasoning)**：結合地理信息（GIS）或先驗知識（如：禁飛區、天氣影響）。

- 3. 實例說明
> **自動駕駛場景**：
> * **L1（目標評估）**：檢測到前方 50 米有一個行人，左側 10 米有一輛巴士。
> * **L2（態勢評估）**：觀察到行人正快速向巴士站移動，且巴士正在進站減速。系統推斷：「行人極大機率準備橫穿馬路趕巴士」。

- 5. L1 與 L2 對比表

| 特性 | L1: 目標評估 | L2: 態勢評估 |
| :--- | :--- | :--- |
| **處理對象** | 單個實體（點跡、航跡） | 實體集合、背景環境 |
| **主要任務** | 狀態估計（位置、速度、身份） | 意圖識別、佈局分析、關係推理 |
| **輸出結果** | 目標清單 (Track List) | 態勢圖 (Situation Map) |
| **數學/邏輯工具** | KF, EKF, GNN, JPDA | 貝葉斯網絡、本體論、圖匹配 |
## L3: 影响评估 (Impact Assessment)核心逻辑：解决**“可能发生什么？”的问题。
它是在 L2 态势的基础上，进行前瞻性预测**。关键任务：威胁评估：分析目标的敌意、致命性以及对我方目标的威胁程度。意图识别：预测目标的行动路径和最终目的。脆弱性分析：识别我方防御或计划中的薄弱环节。应用举例：自动驾驶：基于 L2 判断出“行人要过马路”，L3 进一步评估行人是否会撞上本车，并预测潜在的碰撞时间。数学工具：隐马尔可夫模型 (HMM)、博弈论 (Game Theory)、态势投影。
## L4: 过程评估 (Process Assessment)核心逻辑：解决**“融合得好不好？”以及“如何优化？”的问题。
这是一个元融合（Meta-Fusion）**过程。关键任务：性能监控：评估当前融合结果的精度（如协方差矩阵 $P_k$ 是否收敛）。传感器管理：根据任务需求动态分配资源（如：发现雾大时，命令红外传感器增强扫描，减少激光雷达权重）。反馈控制：根据评估结果调整前三层处理的参数。应用举例：温室控制系统：如果光强传感器失效导致融合数据异常，L4 逻辑会识别出故障，并自动切换至基于时间的默认控制策略或加大其他传感器的信任度。数学工具：信息增益、协同优化算法、多目标规划。
