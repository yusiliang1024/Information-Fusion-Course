# 信息融合：检测融合策略深度笔记

## 1. 概述
**检测融合（Detection Fusion）**是指将多个传感器的观测数据或判决进行综合，形成比单一传感器更完全、更准确的判决。
* **核心任务**：在 $H_0$（目标不存在）与 $H_1$（目标存在）之间做出最优选择。

## 2. 系统拓扑结构
* **分布式并行结构**：局部决策 $u_i$ -> 融合中心 DFC（最常用）。
* **分布式串行结构**：信息按顺序传递并不断修正。
* **树形/分散结构**：多层级或无中心化融合。

---

## 3. 基础逻辑融合准则（硬决策）
这类准则不考虑传感器自身的概率特性（$P_d, P_f$），实现简单但不够精确。

| 准则 | 逻辑描述 | 概率特性变化 |
| :--- | :--- | :--- |
| **“与”准则 (AND)** | $\prod u_i = 1$ | $P_f \downarrow$ (极低), $P_d \downarrow$ (保守) |
| **“或”准则 (OR)** | $\sum u_i \geqslant 1$ | $P_d \uparrow$ (极高), $P_f \uparrow$ (灵敏) |
| **K-out-of-N** | $\sum u_i \geqslant k$ | 灵活折中，由 $k$ 值决定偏向 |

---

## 4. 统计检测准则：从 MAP 到 Bayesian

### 4.1 最大后验概率 (MAP) 准则
**核心思想**：选择使后验概率 $P(H_i|Y)$ 最大的假设，旨在最小化**平均错误概率**。

* **判决表达式 (LRT)**：
    $$\Lambda(Y) = \frac{P(Y | H_1)}{P(Y | H_0)} \underset{H_0}{\overset{H_1}{\gtrless}} \frac{P(H_0)}{P(H_1)}$$
* **适用场景**：已知先验概率 $P(H_i)$，且认为“虚警”和“漏检”的危害等同。
* **退化情况**：若 $P(H_1)=P(H_0)=0.5$，则门限为 1，变为**最大似然 (ML) 准则**。

### 4.2 Neyman-Pearson (N-P) 准则
**核心思想**：在限定虚警率 $P_f \leqslant \alpha$ 的前提下，使检测概率 $P_d$ 最大。

* **判决表达式**：
    $$\Lambda(Y) \underset{H_0}{\overset{H_1}{\gtrless}} \lambda_{NP}$$
* **门限确定**：$\lambda_{NP}$ 是通过方程 $\int_{\lambda_{NP}}^{\infty} p(\Lambda|H_0) d\Lambda = \alpha$ 解出的。
* **优势**：**不需要知道先验概率**，非常适合雷达探测等背景噪声已知但目标出现概率未知的场景。



### 4.3 贝叶斯 (Bayesian) 准则
**核心思想**：引入**代价因子 (Cost Factor)**，最小化平均风险（Bayes Risk）。

* **代价定义**：
    - $C_{10}$: 虚警代价（$H_0$ 为真却判为 $D_1$）
    - $C_{01}$: 漏检代价（$H_1$ 为真却判为 $D_0$）
    - 通常假设正确判决代价为 0 ($C_{00} = C_{11} = 0$)。
* **贝叶斯判决门限**：
    $$\lambda_{Bayes} = \frac{P(H_0)(C_{10} - C_{00})}{P(H_1)(C_{01} - C_{11})}$$
* **推导意义**：如果漏检的代价远大于虚警（如漏掉导弹的后果比误报严重），门限会**变小**，系统变得更灵敏。

## 5. 分布式检测中的对数似然 (Log-LRT)
为了在 GitHub/Markdown 中清晰记录算法实现，建议使用对数求和形式：

若各传感器独立，融合中心的全局判决为：
$$L = \sum_{i=1}^{n} w_i \cdot u_i \underset{H_0}{\overset{H_1}{\gtrless}} \ln(\lambda)$$

其中每个传感器的权重 $w_i$ 取决于其性能：
* 当 $u_i = 1$ 时，$w_i = \ln \frac{P_{d,i}}{P_{f,i}}$
* 当 $u_i = 0$ 时，$w_i = \ln \frac{1 - P_{d,i}}{1 - P_{f,i}}$

## 6. 实战应用：概率获取与权重计算

### 6.1 参数来源
1.  **先验 $P(H_1)$**：历史统计数据（如该海域以往出现潜艇的频率）。
2.  **性能参数 ($P_d, P_f$)**：厂家标定的 **ROC 曲线** 或 离线测试数据。
3.  **代价因子 ($C_{ij}$)**：由业务需求决定（如：误报罚款 100 元，漏报损失 10000 元）。

### 6.2 案例：多传感器防盗报警 (Bayes 视角)
| 传感器 | $P_d$ | $P_f$ | 权重贡献 (报 1 时) |
| :--- | :--- | :--- | :--- |
| **红外 (IR)** | 0.95 | 0.05 | $+2.94$ |
| **震动** | 0.90 | 0.10 | $+2.20$ |

**场景分析**：
- 如果你更在乎“别被小偷搬空”，设置 $C_{01}$ 极大，门限 $T$ 会下移。
- 即使只有一个传感器报警，权重（2.94）可能就足以超过下移后的门限，触发报警。

## 总结对比
| 准则 | 所需信息 | 优化目标 |
| :--- | :--- | :--- |
| **ML** | 条件概率 | 似然最大化 |
| **MAP** | 条件概率 + 先验概率 | 最小化错误概率 |
| **N-P** | 条件概率 + 虚警上限 | 限制虚警，最大化检测 |
| **Bayes** | 条件概率 + 先验 + 代价因子 | 最小化平均风险 |
